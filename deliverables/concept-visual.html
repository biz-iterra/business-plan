CONCEPT DRAWING SHEET A-00 / 表紙 2026.06
事業コンセプト設計図 — DRAFT v1

横に立って伴走し、
一緒に成果を出し、一緒に強くなる。

代わりに全部やってあげる会社ではない。現場でがんばってる人の隣に立って一緒に手を動かし、その人の中にあるポテンシャルを引き出して、気づいたら相手も自分も前より強くなってる——そういう会社の設計図。

伴走
A-01
理念の五本柱 — 土台
基礎構造 / 確定

すべての判断は、この五本の柱の上に建つ。

原体験(学んだ技術が現実で機能し、相手が輝き、その相手から認められた経験)から掘り出したもの。借り物ではない。

P-1何のための会社か

現場でがんばってる人の隣に立ち、一緒に手を動かす。その人が自分でできるようになるまで伴走する。相手も自分も、前より強くなる。

P-2誰のための会社か

顔の見える相手。目の前で困ってる担当者と、その人を抱える組織。「あの人なら任せられる」という信頼が一人ずつつながっていく関係。

P-3自分の強み

噛み合っていない構造を根っこまで掘り、筋の通った形に作り直せること。学んだことを知識で終わらせず、現実でちゃんと動かせること。システムも、業務も、人の配置も、同じ目で見る。

P-4エンジン(動機の核)

自分の力が現実で機能し、担当者が主役として輝き、その担当者から評価される——二段構えの「強い承認」。だから相手を主役にするのは、道徳ではなく欲望と一致した戦略。

P-5迷ったときに自分に聞くこと

  • 全体として筋が通ってるか?
  • 現実でちゃんと機能するか? 根っこに効いてるか?
  • 相手が主役か? 対等に向き合えてるか?
  • これをやって、自分も伸びるか?
A-02
絶対にしないこと — 構造の輪郭
禁止構造 / 確定

やらないことが、この会社の形を決める。

禁
相手を「自分なしじゃ回らない」状態にしない楽だけど、それは相手のためにならない
禁
見た目だけ整えて、実際は動かないものを納品しない成果は現実で機能したかで測る
禁
相手の手柄を横取りしない主役はあくまで相手。喝采は相手に渡す
禁
お金や情など、単一の側面だけで仕事を決めない全部ひっくるめて筋が通っていればいい。100%純粋な案件は存在しない
禁
スキルが高い=偉い、にしない社内でも社外でも同じ。媚びず、見下さず、対等。偉さは現実の成果と筋で決まる
A-03
コンセプトの転換 — 動力機構
回転構造 / v2
育成 → 卒業 ▶▶ 引き出す → 回す → 自走

「育成」は縦の関係、「卒業」は関係の終わり。どちらも対等の原則・信頼の連鎖と矛盾していた。代わりに、相手の中にすでにあるものを引き出し、学ぶ→試す・実践する→振り返る・可視化するの螺旋を回す。円ではなく螺旋——一周ごとに少し高いところへ戻る。その積み上がりが成長で、螺旋が相手の力で回り出した状態が自走。関係は終わらない。

図 A-03.1 / 成長螺旋機構
  • 学ぶ — 現状の構造を一緒に解剖する。「なぜこうなってるか」を担当者自身が理解する。
  • 試す・実践する — 一緒に手を動かして試す。失敗していい。最初はこちら7:相手3、徐々に逆転させていく。
  • 振り返る・可視化する — 振り返る対象は現実の行動。ポテンシャル(得意・苦手)をレンズに解像度高く振り返り、「動いた仕組み」と「できるようになったこと」を見える形で残す。
  • 成長と自走 — 一周ごとに同じ場所ではなく、少し高いところに戻る。その積み上がりが成長。螺旋が相手の力で回り出したら、もう依存ではない。関与は減るが、関係は続く。
A-04
二層構造 — 個人のループ × 組織の配置
断面図 / v1

組織に売って、個人に効かせる。

toB か toC かは偽の二択。買い手は経営者・組織、受益者は担当者個人。お金は組織から流れ、意味は個人に届く。心の置き場所(個人)と財布の向き(組織)は分けていい。

個人の層ENGINE
学ぶ→試す・実践する→振り返る・可視化するの螺旋を一緒に回す。これが価値の本体。担当者が力をつけ、組織の内外から頼られる存在になる。振り返りの精度を、ポテンシャルというレンズが支える。
組織の層FOUNDATION
ポテンシャル診断による共通言語と配置。人と人の対話を円滑にし、チームの体制・配置をきれいにする。個人のループが回りやすい地面を整える役割。効いたかどうかは「対話が円滑になったか・配置が機能したか」という現実で検証する。
設計上の注意: 診断は前菜であって主菜にしない。対話と振り返りのきっかけに限定し、配置の最終判断は現実の行動・成果データで行う。「形だけのものを出さない」原則を診断自身にも適用する。
A-05
商品の骨格 — プログラム型伴走
平面図 / 設計中

売るのは時間ではなく、「状態の変化」。

「月額×稼働日数」ではなく「◯ヶ月で、動く仕組みと自走できる担当者を残すプログラム:一式◯◯円」。中身は伴走でも、売り物は到達点。時間管理は内部の原価管理に格下げする。

PHASE 1 / 学ぶ

構造の解剖

業務フロー・IT・人の噛み合わせを一緒に解剖し、筋の通った改善マップを描く。

PHASE 2 / 試す・実践する

一緒に作る

GAS・AppSheet・生成AIなどで実装。失敗込みで試しながら、担当者の手を動かす比率を段階的に上げる。

PHASE 3 / 振り返る・可視化する

月次レビュー

①現実に動いた仕組み ②担当者ができるようになったこと——成果を二重に可視化。

PHASE 4 / 自走

関与を減らす

週2日→週1→月数回スポットへ。"依存させない"を契約構造そのもので証明する。

A-06
収益の階段 — 時間売りからの離脱経路
階段断面図 / v1

「法人格を持ったフリーランス」で終わらないための断面図。

初日から時間売りを完全に消すことはできない。設計すべきは時間売りの否定ではなく、時間から離脱していく階段。売上の主語が「自分の時間」から「プログラム・資産・関係」へ段階的に移る。

  • L-1プログラム型「6ヶ月で仕組み+自走する担当者を残す:一式◯◯円」。到達点に値段をつけ、人月の心理から抜ける。現金の主柱。
  • L-2自走後のライト契約関与が減る設計が、低稼働×継続課金の層を自動的に生む。10社自走すれば、ほぼ稼働ゼロの固定収益が10本。
  • L-3資産の再利用診断手順・テンプレ・スキル群を社内資産化しながら納品する。同じ価格で稼働が半分になる——一人会社の実質的な利益エンジン。
  • L-41対N講座・ワークショップ・診断ツールなど、1時間がN社に届く形。映像・イベント人材のネットワークが活きるのはここ。
A-08
商品と価格の階段 — 入口から継続まで
価格構造 / DRAFT

入口は割る。本体は割らない。

無料相談は「診断」ではなく対話と相互選別の場。有料診断は"深さ"で2段(規模ではない)。伴走期は時間でなく状態に課金する月額固定。割引は立ち上げ期の入口だけに限定し、本体月額は最初から通常価格で出す。

00

無料相談 ¥0

診断ではない。何に困っているか、螺旋を回す相手がいるかを対話で見極める。ここはあなたが「断る」ための選別の場でもある。

01

構造診断・竹(標準) ¥300,000 ★本命

主要1業務領域の構造診断(2〜3週間)。納品は3点セット=現状構造図(As-Is)／改善マップ(To-Be＋優先順位)／伴走計画書(期間・体制・金額・自走の基準)。単体で買う価値が成立する成果物。

02

構造診断・松(全社) ¥500,000〜

全社横断・部門間の連携まで診る。規模が大きい会社は自然にこちらが必要になる。「規模で売る」のではなく「診る深さ」で分ける。

03

濃い伴走期 月額固定 / 診断後に確定

シナリオA準拠(週2日相当・月70〜80万を軸)。期間は固定せず、長さは診断で会社ごとに見積もる。終わりを決めるのはカレンダーでなく自走の基準。前払いの月額固定(準委任)——時間でなく状態の変化に課金する。工数は内部の原価管理に格下げ。

04

逓減期 月額(縮小)

関与が減っていくことを契約構造そのものに書く。「依存させない」を口約束でなく数字で証明するフェーズ。

05

ライト契約 月額 ¥50,000〜100,000

自走後の継続。実務は持たず相談・保守ベース(月◯回レビュー＋チャット相談)。実装が要る時は別途スポット見積もり。"安売りの何でも屋"にしない枠を最初から引く。

立ち上げ枠
先着5社・◯月まで:構造診断 ¥100,000(通常30万)。条件=事例公開の許可／率直なフィードバック／良ければ紹介1社。無料にせず1円でも取り、相手を当事者にする。伴走期の月額は割らない(本体の相場が記憶として固定されるため)。これは売上でなく、実績と信頼連鎖への投資。
