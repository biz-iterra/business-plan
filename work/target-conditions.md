# ターゲット再定義 — 業界絞りから「顧客条件」へ（コンサル2セッションの整理）

> 作成: 2026-07-03 / ステータス: DRAFT v1（たたき台。本人が違和感を起点に削る前提）

> 位置づけ: **ターゲット定義の現行の正。** 旧・業種ベースの成果物（介護＋小売二本柱：target-definition / persona / persona-retail / model-case ほか）は `archive/` へ移動（§7の対応表）。決定の経緯は `docs/HANDOFF.md` 13章。

## 0. 決定（本人・2026-07-03）

**一次ターゲットを「業界（介護・小売）で絞る」のをやめ、「一定の条件に合っている中小企業」に切り替える。**

- **ターゲットの一言キーワード（本人）＝「経営課題にITを活用することに、理解のある企業」。** 条件M1〜M4（§2）はこの一言の分解。対外の言葉としても使える（業種を言わずにターゲットを名指しできる）。
- **本人の補足（同日）**：介護・小売のターゲットを**完全になくすのではなく、そこに絞りすぎないようにする**ということ。archive/ の業種資産は捨てたのではなく手札（§2 業種の扱い）。

- コンサルの評価：業界による一律のセグメンテーションは現時点では時期尚早・経済的合理性が低い。
  - 小売：可能性はあるが、単価・継続率の面で主軸化は慎重を要する。
  - 介護：ニーズは非常に強いが、現場の負担・予算制約・導入定着のハードルが極めて高く、収益化の効率が悪い。業界特有の構造的疲弊・低リテラシーに巻き込まれると、一人会社のリソースが泥沼化する。
- 判断の軸を「業界」から「**顧客の内部環境（準備状況）**」へ移す。

## 1. コア・コンセプト：「コーポレートIT」

**「IT担当者が不在、あるいは詳しくない中小企業の混乱を整理し、必要な部分だけを前に進める支援」**

- 単なる「IT支援」という曖昧な役務でも、便利屋でもなく、経営課題をITで整える「**コーポレートIT**」基盤の構築支援＝ポジション特化の立ち位置。
- 「IT担当者不在」「業務の属人化」というボトルネックを、属人的な善意ではなく**商品**として解消する。
- 3段階のリテンション構造（→ §4：既存の商品の階段とほぼ一致。商品構造は生きる）：
  1. **診断フェーズ**：現状整理とロードマップ。専門性の誇示を排し「見える化」で経営者に理解＝安心を提供。
  2. **実行フェーズ**：月額伴走とツール導入。属人化という負債を清算し、システムを企業の資産に変える。
  3. **定着フェーズ**：運用ルールと可視化＝ITガバナンスの確立。ここで代替不可能な「経営インフラ」になる。

## 2. 顧客選定条件（新ターゲット定義の核）

### 受注条件（Must-Have）— 全部そろっているのが理想形
| # | 条件 | 何を見るか |
|---|---|---|
| M1 | **経営層がIT整備の必要性を認識し、予算を確保している** | 「そのうち」でなく投資として認識しているか。値切り前提でないか |
| M2 | **社内に共に動く担当者がいる（丸投げ体質でない）** | 螺旋を一緒に回す相手が実在するか。※従来の「担当者が1人いる会社」と同じ核 |
| M3 | **組織が変化の過程にある**（成長・承継・M&A・店舗展開 等） | 変化＝痛みが顕在化するトリガー。※従来の「刷新・退職・DX焦り」の一般化 |
| M4 | **今すぐ整える理由（緊急度）がある** | 「そのうち」案件は追わない |

### 排除条件（Red Flags）— 1つでも濃く出たら断る関門
| # | 条件 | なぜ排除か |
|---|---|---|
| R1 | 便利屋として安く使おうとする／値切り前提の予算感 | 対等でない・本体単価が崩れる（立ち上げ枠の原則とも矛盾） |
| R2 | 丸投げ体質 | 「依存させない」原則と正面衝突。自走の相手がいない |
| R3 | 意思決定者が不在で、現場が非協力的 | 診断しても実行に進めない構造 |
| R4 | IT整備が追加の業務負担としか捉えられておらず、改善意欲が低い | 教育コストが収益を上回る。定着しない |

### 判断指標の追加：「身体感覚」
- 案件選定は「単価・工数」だけでなく、**「その仕事を終えた後に、自分の身体が軽くなるか、喜びを感じるか」**を持続可能性の指標として使う。
- 「身体が重くなる」案件は、例外なく隠れたコストや精神的摩耗を伴い、中長期の収益性を悪化させる——という運用。
- 位置づけ：P-5（判断基準）の4項目を置き換えるものではなく、**5つ目のチェック**として足す。言葉は本人が普段の言い方に直す前提。

### 業種の扱い（誤解しないための注記）
- 介護・小売を**排除するのではない**。「業種で決め打ちしない」だけ。条件（M1〜M4）に合う会社が介護や小売なら普通に受ける。
- 蓄積した介護・小売の業種知識（archive/ の persona・model-case 等）は、その業種の案件が来たときの手札として温存。

## 3. 何が変わって、何が変わらないか

### 変わる（上書き）
| 項目 | 旧（〜2026-06-24） | 新（2026-07-03〜） |
|---|---|---|
| 一次ターゲット | 介護＋小売（独立系中堅）の二本柱 | 条件に合う中小企業（業種不問） |
| ターゲットの単位 | 業種 × 規模 × トリガー | 顧客の内部環境（準備状況）M1〜M4 |
| ペルソナ | 介護二層／小売ICP＋3ペルソナ | 業種固有ペルソナは一旦棚上げ（archive） |
| クロージングの数字 | 特養C・HCチェーンHのモデルケース | **業種固有数字は未確定に戻る**（→残る宿題 §6） |
| リード獲得の場 | 介護の業界団体・介護ベンダーアライアンス等 | **再設計が必要**（→残る宿題 §6） |

### 変わらない（無傷で残る）
- **理念5本柱・絶対にしないこと5つ**（HANDOFF 1〜2章）。むしろ Red Flags は「依存させない・対等」の裏返しで、既存原則と完全に整合する。
- **商品の階段**：無料相談→Phase0診断→濃い伴走→逓減→ライト。コンサルの3段階（診断→実行→定着）はこれとほぼ同型（§1）。
- **課金形態**：月額固定（準委任）一択。工数×時給はやらない。
- **3社上限・立ち上げ枠**の方針。
- **対外サイトで業種名を出さない方針**（web-structure §0）。もともと業種非公開だったので、サイト・LP・メソッド関連の成果物は今回の転換でも生きる。
- **補助金フック不可（助成金のみ）・紹介は保険**（HANDOFF 11章）。

## 4. サービスモデルの対応関係（コンサルの言葉 ↔ 既存設計）

| コンサルの言葉 | 既存設計 | ズレ |
|---|---|---|
| 診断フェーズ（現状整理とロードマップ） | Phase0 構造診断（As-Is／To-Be／伴走計画） | ほぼ一致 |
| 実行フェーズ（月額伴走とツール導入） | 濃い伴走期（月額固定・準委任） | ほぼ一致 |
| 定着フェーズ（運用ルール・可視化・ITガバナンス） | 逓減期→ライト契約 | **「ITガバナンス確立」という言語化は新しい**。ライト契約の中身を「保守・相談」から「ガバナンス維持」へ言い換えられる可能性 |
| 「代替不可能な経営インフラになる」 | L-2 自走後のライト契約 | ⚠️ 表現に注意。「代替不可能」は**依存させない原則と言葉の上で衝突**。実態は「自走した上で、いてくれると安心」の関係を目指す（本人の言葉で言い直す） |

## 5. その他のコンサル指針（事業設計に効く範囲のメモ）

1. **人員計画＝WAIT**：サービスの型が固まる前の増員・固定外注・営業代行は管理コスト先行で経営を圧迫。まず「本人にしかできない判断」と「外部に渡せる作業」を分解し、月額契約が確保できてからスポット補助を導入（3〜6ヶ月目安）。→ 既存の「協力者の稼働を前提に商品を設計しない」（HANDOFF 7章）と整合。
2. **保険解約金（海外資金）の配分**：50〜70% 安全資金（心理的レバレッジ＝悪質案件を断れる余裕）／20〜40% 生産性投資（PC等の仕事道具）／5〜10% 検証投資（小規模テスト営業・回復・交流）。大規模広告・根拠のない人件費・生活基盤の急変（引っ越し等）は回避。
3. **メディア**：発信は信頼形成に有効だが、当初から有料コンテンツ販売は狙わない。「難しいことを現場目線で整理してくれる会社」という認知を先に作る。YouTube は相談窓口でなく人柄・説明力を伝える**補助媒体**。→ 既存の「オウンドメディア＝信頼装置＋種まき、主軸にしない」（HANDOFF 11章③）と一致。
4. **イベント出展は非推奨**：不特定多数への露出は効率が悪い。紹介・個別相談・登壇など質の高い接点を優先。→ 既存の登壇方針と一致。
5. **未収金（座株式会社）**：少額訴訟を含め請求は正当。感情・人格評価・制裁意識を排し、「契約合意の証拠／納品事実／請求額と期限徒過／最終催告後の未払い」に論点を絞る。位置づけは資産防衛（未収金＝許可なき値引き）＋経営者としての実務トレーニング。※事業設計の外だが記録として残す。
6. **サービスの型化を1ヶ月以内に**：初回診断・改善ロードマップ・月額伴走のパッケージ明文化＋受ける/受けない条件のリスト化。→ 前者は `service-definition.md` でほぼ済み。後者が本ドキュメント §2。
7. **拡大より構造の安定**：短期の規模拡大・広告大量投資・性急な増員はしない。「原点回帰」＝これまでの経験の再統合を軸に、持続可能な収益基盤を先に作る。

## 6. 媚びない指摘（この転換の弱点と残る宿題）

率直に書く。同意できない点は同意しない、の原則どおり。

1. **「条件ベース」はリード獲得の絞りに使えない。** M1〜M4 は「会って話して初めて判る」条件が多い（予算の有無・丸投げ体質・担当者の実在）。業種で絞っていた最大の理由は「どこに行けば会えるか・何を発信すれば刺さるか」を決めるためだった。条件ベースに切り替えた瞬間、**登壇テーマ・発信内容・アライアンス相手を選ぶ軸が消える**。→ リード獲得設計はやり直し。入口は「業種」でなく「状態・トリガー」（M3：承継・M&A・店舗展開・成長期）で切る等のたたき台が次の宿題。※M&Aアドバイザーの人脈（HANDOFF 7章）は M3 と直結する入口候補になり得る。
2. **介護外しの根拠は実データではない。** HANDOFF 4章の覆さない条件は「一次を動かすには実データが要る」だった。今回の転換の根拠はコンサルの評価＋本人の納得であり、介護で実際に営業して駄目だった実績ではない。本人が納得して決めたのだから上書きは成立するが、**「介護は収益化の効率が悪い」はまだ仮説**であることは記録しておく（条件に合う介護案件が来たら普通に受ければ検証できる）。
3. **モデルケースの数字が宙に浮いた。** クロージング・LP⑦・定量シートの源泉だった特養C／HCチェーンHは業種固有。業種を外した営業トークで使う「想定モデル」の数字は**未確定に戻った**。業種非依存の汎用モデルケース（例：多店舗/多拠点の中小企業）を作り直すか、業種別数字を「相手に合わせて出す手札」として使い分けるか、要決定。
4. **「代替不可能な経営インフラ」という言葉は要注意**（§4）。依存させない原則との整合は、言葉を本人が引き取るときに必ず調整する。

### 覆さない条件（この決定を業種絞りに戻すには）
- 条件ベースで動いて「入口が作れない・リードが取れない」実データが出る、**または**特定業種で連続受注し集中の合理性が実証される——のどちらか。コンサルの一言だけでの再反転はしない（往復コストが大きい）。
