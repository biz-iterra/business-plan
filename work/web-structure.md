## 制約（v1から継承・変更なし）

- **特定の業種名（介護・小売 等）・ペルソナ名・モデルケースの固有名は出さない。** 課題は「状態・構造」で括る。※条件ベースになった今、この制約は戦略と完全に一致した（隠す理由すらなくなった＝そもそも業種で選ばない）。
- 言葉は仮置き＝**本人が口調で書き換える前提**。
- サイトの役割＝**集客の主軸ではなく「出会った相手が見て安心する信頼装置＋無料相談への一歩」**（変更なし。lead-acquisition-v2 のチャネルA〜Eで会った人の受け皿）。
- 事例ゼロ期は「想定モデル」と正直に明示（2章②）。

## 1. サイトマップ（9ページ・骨格はv1継承）

```
トップ（コーポレートIT ＝ 一言の約束＋選別の入口）
├─ サービス（コーポレートITの3段階 ＝ 診断→伴走→自走）
├─ メソッド（ITERRAメソッド ＝ 3軸×3フェーズ・4つの問い・協業性）
├─ 合う会社・合わない会社（条件の公開 ＝ 相互選別）      ← v1「こんな会社へ」を拡張
├─ 料金の考え方（診断は金額明示▶要判断・伴走はレンジ）
├─ わたしたちについて（理念・しないこと・代表）
├─ 想定モデル（費用対効果の一般形。数字は汎用ケース確定後）
├─ ジャーナル（発信・信頼装置・種まき）
└─ 無料相談（唯一のCV・30分・相互選別）
```

**回遊の設計**：トップ →（うちのことだ）→ 合う会社・合わない会社 or メソッド →（納得）→ 無料相談。料金・わたしたちは不安を消す補助。**どのページも出口は無料相談1つ**。

## 2. 各ページ構成

### ① トップ
- **目的**：3秒で「誰の・何の会社か」＋「客を選ぶ会社だ」と伝わること。
- **ブロック**（6ブロック＝v1互換。実コピーは `site-copy-draft.md` Part1）：
  1. **FV**：一言の約束（4案から本人選択）。サブに **「経営課題にITを活用したい中小企業の、コーポレートIT部門を一緒に立ち上げ、自走まで伴走する」** 系の説明を検討（※コーポレートITという語を最初に定義する場所）。
  2. **課題の共感**：状態で括る（道具が使われない／詳しい人が1人だけ／バラバラ導入／ベンダーに言いなり／経営と現場の言葉が噛み合わない）＋**変化の入口**（社長が代わった・会社を買った/買われた・拠点が増えた・人が急に増えた）を追加。
  3. **だから何をするか**：引き出す→回す→自走 を1行ずつ。
  4. **他のIT会社と何が違うか**：3軸（→ITERRAメソッドへ）。
  5. **しないこと**：依存させない／形だけ出さない／手柄を取らない。
  6. **CTA**：無料相談30分。「売り込みではなく相互選別」。
- **v2の追加注**：ブロック2に「変化の入口」を足す以外、v1の骨格・コピーはそのまま使える。

### ② サービス（コーポレートITの3段階）
- **目的**：商品の全体像。「コーポレートIT＝経営課題をITで整える社内機能。それが無い会社に、外から立ち上げて、最後は会社のものにして渡す」という筋を通す。
- **ブロック**：
  1. **コーポレートITとは**：情シスとも、ITコンサルとも、開発会社とも違う——経営とITの間を整える機能。多くの中小企業に「無い」もの。
  2. **① 引き出す＝構造診断**：4週間・納品4点（現状構造図／改善マップ／伴走計画／経営メリット定量シート）。**単体で完結する**（診断だけで終えてよい）ことを明記＝関門の公開。
  3. **② 回す＝伴走**：月額固定・接点と状態で約束（「週n日」を書かない）。何を一緒にやるかは `service-operations.md` §3 の月次ルーチンから対外語訳（定例で螺旋を回す／ベンダー折衝に同席し翻訳する／経営者に月次で数字を報告する）。
  4. **③ 自走＝逓減→ライト**：自走の基準で手を放す。**「自走したら関係が切れる」のではなく「ITガバナンスの維持役として残る」**（コンサルの「定着フェーズ」の言語化を採用。※「代替不可能なインフラ」という言い方はしない＝依存の匂い）。
  5. **時間でなく状態を売る**：課金の考え方1行＋料金ページへ。
- **注**：手順の詳しさは service-operations の対外要約に留める（手の内を全部晒さない。在り処まで＝sales-process 原則）。

### ③ メソッド（ITERRAメソッド）
- **目的**：事例ゼロ期の主役。「やり方」で信頼を取る。実コピーは `site-copy-draft.md` Part2（§A〜F・v2互換）。
- **ブロック**：§A ITERRAメソッドとは（**必ず一行説明とセット**：頭脳×実務×情緒の3軸を、引き出す→回す→自走の3フェーズで使い切る）／§B 3×3マトリクス／§C **4つの問い**（生命線・業種非依存）／§D なぜこのメソッドか（既存IT会社の限界＝役割類型で）／§E 協業性（**マージンを取らない**＝顧客の側に立つ担保）／§F 診断への橋渡し。

### ④ 合う会社・合わない会社（★v2の新しい核）
- **目的**：条件ベースのターゲティングを**そのまま対外に出す**。読者に「うちは合う側だ」と自己判定させ、合わない客を入口で減らす（断る関門をサイトが先にやる）。
- **ブロック**（`target-conditions.md` §2 の対外語訳・たたき台）：
  - **リード**：「私たちは、お手伝いできる会社が限られています。**同時に3社まで**しかお受けしません。だから、正直に書きます。」（3社上限を弱点でなく品質保証として公開）
  - **力になれる会社**：
    - 経営課題にITを活用することに、経営者自身が理解と関心を持っている
    - 社内に「自分たちで良くしたい」と思っている人が、1人でもいる
    - 会社が変わり目にある（代替わり・合併・拠点や人の急拡大・事業の転換）
    - 「そのうち」ではなく、今、整えたい理由がある
  - **力になれない会社**：
    - 全部お任せしたい（丸投げしたい）——私たちは代行会社ではありません
    - ITは安く済ませたい・とりあえず値段だけ知りたい
    - 決める人が出てこない（現場だけ・窓口だけで進めたい）
  - **締め**：「合わない、と感じたら申し込まないでください。合いそうなら、30分だけ話しましょう。」→ CTA
- **媚びない注**：ここは**強い選別トーンなので、口調の調整が最重要ページ**。事実（対等・非依存）を保ったまま、感じの悪さ（上から目線）に落ちない言い換えを本人がやる。「力になれない会社」を公開すること自体の可否も本人判断（▶要判断。出さない場合は「力になれる会社」だけでも機能する）。

### ⑤ 料金の考え方
- **目的**：金額で脱落させず、安売りにも見せず、「誠実に決めている」と伝える。
- **ブロック**（`pricing.md` と接続）：
  1. **考え方**：時間を売らない（月額固定）／値引きしない／会社の状態に合わせて診断で確定する。
  2. **構造診断＝◯◯万円（税別）〜**：**金額明示を提案（▶要判断・v1からの変更）**。理由：入口商品・単体で完結・比較検討の土俵に乗るには価格が要る。大型（多拠点等）は個別見積と併記。
  3. **伴走＝月額固定・レンジのみ**（例「月◯◯万〜◯◯◯万。診断で確定します」）：一覧は出さない（v1どおり。相場固定を避ける＝既決）。
  4. **自走後＝ライト（月◯万〜）**：関係の続き方として。
  5. **含まれないもの・外注の扱い**：実装は専門家と分業し**実費を開示、中間マージンを取らない**（③§Eと同じ話を財布の面から）。
- **注**：立ち上げ枠（先着5社・診断割引）をサイトに出すか：**出さない**を推奨（安く見える・期限管理が発生。営業の場で個別に出す）。▶要判断。

### ⑥ わたしたちについて
- **目的**：一人会社の不安を人と哲学で埋める（v1どおり）。
- **ブロック**：理念（横に立って一緒に成果を出し、一緒に強くなる）／**絶対にしないこと5つ全文**／代表プロフィール（**実働者であること**＋経歴。「経営とITの間の翻訳を、現場で手を動かしながらやってきた」筋）／会社概要。
- **v2の追加**：**「なぜ業種を絞らないのか」への答えをここに置く**（聞かれる想定問答）：「業種より、会社の状態で決めています。同じ業種でも、合う会社と合わない会社があるからです」——条件ベース戦略の対外的な言い分をこの1問に集約。

### ⑦ 想定モデル
- **目的**：費用対効果を具体で見せる。初期は「想定モデル」と明記した一般形。
- **ブロック**：曖昧なシステム導入の出戻りコスト → 診断で回避／運用改善で年◯万——の一般形。**数字は汎用モデルケース（HANDOFF 13章 宿題②）の確定待ち。それまでこのページは公開しない**（数字なしで公開すると「形だけ」に落ちる）。
- 1社目ができたら実事例に差し替え（v1どおり）。

### ⑧ ジャーナル
- **目的**：種まき＋出会った後の安心（v1どおり・主軸化しない・KPIは相談につながった数のみ）。
- **テーマ（v2で lead-acquisition-v2 §4 と統一）**：①社長が代わったら、ITが誰にも分からなくなった ②IT担当の求人を出しても来ない ③システムはあるのに経営の数字が出てこない ④AI活用の前に片づける3つのこと（＝4つの問いの読み物化と併走）。

### ⑨ 無料相談（唯一のCV）
- **目的・ブロックはv1どおり**：30分・相互選別・ミニ診断にしない・事前フォーム。
- **v2の追加**：事前フォームに**条件の淡い予選**を仕込む（例：立場〈経営者/担当者/その他〉・会社の変わり目〈代替わり/合併/拡大/特になし〉・決裁の見通し〈自分が決める/決める人に繋げる/未定〉）。M1〜M4を相談前に淡く観測する装置。**足切りには使わない**（相談で確かめるのが本筋）。

1. **本人**：この構成の違和感出し → ④の口調調整・▶要判断3つ（診断金額明示／「力になれない会社」公開／立ち上げ枠非掲載）。

3. **現行の介護特化サイトの扱い（▶要判断）**：リニューアル時に旧コンテンツをどうするか——(a) 全面差し替え（推奨：新ポジショニングと混線させない）(b) 介護コンテンツをジャーナル内の1カテゴリに格下げして残す（過去の縁・検索流入は残る）。**「介護もできる」は隠す必要はないが、トップの顔にはしない。**
4. 想定モデル（⑦）だけ数字待ちで後行公開。他8ページ先行で公開可。

## 4. 媚びない注（v2全体）

- **④を作る勇気がこのサイトの差別化そのもの**。条件ベース戦略の唯一の対外表現がここ。薄めると「どこにでもあるIT支援サイト」に戻る。
- 業種を出さない×条件で選ぶ、は**抽象に逃げやすい構図が v1 より強まった**。打ち消す具体は③の「4つの問い」と②の納品4点、④の自己判定リスト。この3箇所の手触りが生命線。
- サイトは信頼装置（集客主軸でない）の位置づけは変わらないが、**lead-acquisition-v2 のチャネル全部が「会ったあとサイトを見せる」前提**になった。公開の遅れ＝営業着手の遅れに直結する。⑦以外を先に出す。
