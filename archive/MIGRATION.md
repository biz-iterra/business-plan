# Claude Code 移管ガイド

このプロジェクト（ゼロからの事業設計）を Claude Code に移して続けるための手順書。
WSL2 + Claude Code 環境を前提に書いています。

---

## 1. 持っていくファイル（3点 + この手順書）

| ファイル | 役割 | 種別 |
|---|---|---|
| `HANDOFF.md` | **最重要。** 決定事項・各決定の「なぜ」・未決事項・申し送り。新セッションの土台 | 引き継ぎ |
| `concept-visual.html` | 事業コンセプト設計図（9シートの図解。理念〜価格〜工程） | 成果物 |
| `keiei_model.xlsx` | 経営シミュレーション（8シート。前提を変えると全再計算） | 成果物 |
| `MIGRATION.md` | この手順書 | 手順 |

---

## 2. フォルダ構成（推奨）

ローカルに新しいリポジトリを作り、こう置く。

```
iterra-business-design/
├── CLAUDE.md                    # ← Claude Codeが最初に読む。下記テンプレを貼る
├── docs/
│   ├── HANDOFF.md               # 引き継ぎ文書（決定と根拠）
│   └── MIGRATION.md             # この手順書
├── deliverables/
│   ├── concept-visual.html      # コンセプト設計図
│   └── keiei_model.xlsx         # 経営モデル
└── work/                        # ここから先の作業（事業計画書など）を置く
```

---

## 3. セットアップ手順

```bash
# 1. リポジトリ作成
mkdir iterra-business-design && cd iterra-business-design
git init
mkdir -p docs deliverables work

# 2. ダウンロードした4ファイルを配置
#    HANDOFF.md, MIGRATION.md      -> docs/
#    concept-visual.html, keiei_model.xlsx -> deliverables/

# 3. CLAUDE.md を作成（下の §4 の内容を貼る）

# 4. 初回コミット
git add -A
git commit -m "事業設計を移管：理念・コンセプト・商品・価格・経営モデル（DRAFT v1）"

# 5. Claude Code 起動
claude
```

## 4. CLAUDE.md テンプレート（そのまま貼る）

```markdown
# このプロジェクトについて

ゼロベースで「事業の理念・コンセプト・商品・収益構造」を設計している。
作業を始める前に、必ず `docs/HANDOFF.md` を読むこと。特に以下は前提として常に効かせる：

- **判断基準（HANDOFF 1章 P-5）**：迷ったら全体の筋／現実で機能するか／相手が主役か／自分も伸びるか
- **絶対にしないこと（HANDOFF 2章）**：依存させない／形だけ出さない／手柄を取らない／単一の側面で決めない／スキルを偉さにしない
- **棄却済みの選択肢を蒸し返さない**：「育成→卒業」コンセプト、「工数×時給」課金は理由があって却下済み（HANDOFF 3章・4章）

## 進め方のルール
- たたき台を出す → 本人が違和感を起点に削る／直す、の反復で進める
- 媚びない。合理と感情の両面で率直に。同意できない点は同意しない
- 整った対句や借り物の専門用語より、本人が普段しゃべる言葉に寄せる
- 抽象論で終わらせず、根本原因まで掘った具体を出す

## 成果物
- `deliverables/concept-visual.html`：コンセプト設計図（編集時はブラウザ表示で見た目を確認）
- `deliverables/keiei_model.xlsx`：経営モデル（前提シートの黄色セルが入力。数式は壊さない）

## 次にやること（HANDOFF 9章より）
1. 営業ファネルの変換率を実態の数字に寄せる（最優先）
2. 3年目まで会計レベル（税込み）に拡張
3. 役員報酬を上げるタイミングと最適額のシミュレーション
4. 事業計画書（文章版）の作成
5. リード獲得の具体設計（自前の入口・最初の5社）
6. ポテンシャル診断の伴走期での組み込み方
7. 理念・コンセプト文書を「人間味のある言葉」へ書き直し
```

## 5. Claude Code に投げる「最初の一言」例

起動後、最初のプロンプトはこれが効率的：

> `docs/HANDOFF.md` と `CLAUDE.md` を読んで、現状の理解を3行で要約して。
> その上で、未決事項①「営業ファネルの変換率を実態に寄せる」から始めたい。
> まず `keiei_model.xlsx` の営業ファネルシートの今の前提値を確認して、
> 実態を聞き出すための質問を出して。

---

## 6. 引き継ぎ時の注意

- **xlsx を編集するときは数式を壊さないこと。** 前提シートの黄色い入力セルだけを変える。
  検算には `openpyxl` で読み、`recalc`（LibreOffice等）で再計算してから値を確認する。
- **HTML の図は座標を手で合わせない。** A-03 の循環図のように、ノード配置・矢印は
  三角関数で計算して生成する（手調整は崩れる）。編集後は必ずブラウザで表示確認する。
- **決定の「なぜ」を消さない。** 新しい判断をしたら HANDOFF.md に追記し、
  「いつ・なぜ・覆さない条件」を残す。これが次のセッションの筋を守る。
