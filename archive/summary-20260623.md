## 2. 決めたこと（方針）

### 新規顧客獲得（マーケ）— 3つの方針転換
1. **補助金フックは使わない**（採択型制度はトラブル源。助成金のみ可）。勝ち筋「現場の痛みを経営数字に翻訳」は補助金抜きでも生きる。
2. **母はサービスの参考のみ**（顧客紹介・ビジネス接続には使わない）。介護という業種選定自体は維持。
3. **紹介は保険に格下げ**（頼ると破綻する）。1社目から自力が主軸。
- 追加：**公的機関の"場"（よろず／商工会議所）は使う**（補助金とは別物・初速の本命）。発信は**オウンドメディアを検討**（集客主軸でなく信頼装置＋種まき）。

### サービスの実体化 — 固める核＝「商品の中身・契約条件・提供形態」
- **伴走は「週何日」で約束しない**（時給発想回避→接点と状態で定義）。
- **診断は現場観察で最低1回訪問必須**。
- **成果物の権利を分ける**（顧客固有＝顧客／汎用フレーム＝自分。L-3資産化の生命線）。
- **介護＝要配慮個人情報**。NDA・データ扱いを形だけにしない。

### 無料相談の設計の軸
- **ミニ診断にしない**。見せるのは「課題の"在り処"」まで／「解き方」は診断から。
- クロージングは**「まず診断一本」**（伴走を並列で出さない＝階段を壊さない）。筋が通らねば**断る関門**。

## 3. 無料相談からの導線（全体像）

```
HP／オウンドメディアのCTA「無料相談 30分」＋事前フォーム
 → 困りごとを聞く →「なぜ？」で目的に開く
 → 4パターンで“構造問題”に名指し（道具屋に落ちない核＝どれも「業務構造が整理されてない」に合流）
     A AI活用      → 焦り型=エスカレーション／放置型=情報共有
     B 担当退職     → 属人化を要素分解／代行は自ら断る → ライト契約へ
     C システム最適化 → 部署単位導入の連携欠如／ガバナンス
     D 効率化・自動化 → 無くせる作業の自動化／俯瞰役不在＝部署間対立
 → 事例（当面はメソッド）ページで“在り処”を見せる（解き方は渡さない）
 → クロージング：診断＝将来コストの保険（出戻り◯倍）。担当者の評価＋会社のため。
     「一度、診断してみませんか？」（通常30万／最初の5社は10万）
```
※専門用語（BPMN・ガバナンス）は相手のリテラシーで翻訳 or そのままを選ぶ。
