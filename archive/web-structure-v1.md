# Business支援サービスサイト 構成案（DRAFT v1）

> 作成: 2026-06-25 / 役割：ITERRAのビジネス支援（IT伴走）の **事業案内サイト** の構成。`lp-structure.md`（無料相談LP＋メソッドpage）を内包し、サイト全体へ拡張。
> 素材：HANDOFF 1〜6章／`method.md`（3軸×3フェーズ）／`sales-process.md`（4パターンの問い）／`model-case.md`（数字＝固有名を伏せて一般化）。
>
> ## 制約（本人指示・2026-06-25）
> - **特定の業種名（介護・小売 等）・ペルソナ名・モデルケースの固有名は一切出さない。** 課題は「状態・構造」で括る。
> - 営業上の一次ターゲット（業種の絞り）は **内部戦略として非公開**。サイトは普遍的な価値提示に徹する。
> - **メソッド名は仮置き `[メソッド名]`**（別途検討中）。
> - 言葉は仮置き＝**本人が口調で書き換える前提**（整いすぎ・借り物・対句は本人が削る）。

---

## 0. サイトの役割と前提（媚びない注）

- **このサイトは「集客の主軸」ではない。** 経営者は自分から検索して来ない／現場担当者は探す時間がない（HANDOFF 11章③）。サイトの本当の役割は **「登壇・紹介・発信で出会った相手が、見て安心する信頼装置＋種まき」**。初速の集客は別施策（場・登壇）で作る。
- だから構成のゴールは2つだけ：**①「この人なら筋が通っている」と伝わる ②無料相談に一歩踏み出させる**。情報量で殴らない。
- **事例はゼロ**（立ち上げ初期）。架空・想定の効果を出すときは **「想定モデル」と正直に明示**する（2章②「形だけ出さない」を自分のサイトにも適用）。

## 1. サイトマップ（ページ一覧）

```
トップ
├─ サービス（何をする会社か＝引き出す→回す→自走）
├─ メソッド（[メソッド名]＝3軸×3フェーズ・考え方・協業性）   ← lp-structure §B
├─ こんな会社へ（課題＝"状態"で括る・4つの入口）
├─ 料金の考え方（レンジ＋相談で確定）
├─ わたしたちについて（理念・しないこと・代表）
├─ 事例 / 想定モデル（初期は想定ケース→1社目後に差し替え）
├─ ジャーナル（発信・信頼装置・種まき）
└─ 無料相談（CTA・申込フォーム）            ← lp-structure §A のLPがCVの受け皿
```

→ **回遊の設計**：トップ → メソッド or こんな会社へ →（納得）→ 無料相談。料金・わたしたちは「不安を消す」補助。事例は当面"想定モデル"で代替。

## 2. 各ページ構成

> **①トップ・③メソッドの "実際のコピー（文章）" は `work/site-copy-draft.md`（2026-06-28）に肉付け済み。** 本章は構成（どのブロックを置くか）、向こうが言葉。

### ① トップ
- **目的**：3秒で「誰の・何の・なぜ筋が通るか」。深掘りは各ページへ送る。
- **ブロック**：
  1. **FV（ファーストビュー）**：一言の約束。〈道具を売る前に、噛み合っていない構造を直す〉系の骨子（※文言は本人）。
  2. **課題の共感**：道具を入れても定着しない／担当に依存／経営と現場の言葉が違う——**業種を出さず"状態"で**3つほど。
  3. **だから何をするか**：引き出す→回す→自走 の3フェーズを1行ずつ。
  4. **他のIT会社と何が違うか**：3軸（頭脳×実務×情緒）を全部持つ（→メソッドへ）。
  5. **しないこと（輪郭）**：依存させない／形だけ出さない／手柄を取らない（→信頼の核）。
  6. **CTA**：無料相談（30分・相互選別）。
- **注**：FVは本プロジェクト最大の言い換えポイント。借り物コピーは置かない。

### ② サービス（何をする会社か）
- **目的**：商品の全体像＝**3フェーズ＝商品の階段**を理解させる。
- **ブロック**：
  - **引き出す**：構造診断（Phase0）。現状の歪みと、相手の中のポテンシャルを引き出す。**納品4点**（現状構造図／改善マップ／伴走計画／経営メリットの定量シート）。
  - **回す**：濃い伴走（月額固定）。学ぶ→試す→振り返るの螺旋を一緒に回す。
  - **自走**：逓減 → ライト。相手の力で回る状態へ。**依存させず、関係は続く。**
  - **時間でなく状態を売る**：「週何日」で約束しない（HANDOFF 6章・課金の考え方）。
- **注**：ここで "代行屋でない／裏方でない＝対等な並走" を明確に。

### ③ メソッド（`lp-structure.md` §B を流用）
- **目的**：事例ゼロ初期の主役。「やり方」で信頼を取る。
- **ブロック**：
  1. **[メソッド名]とは**（名前確定後に差し込み）
  2. **3軸 × 3フェーズ**のマトリクス1枚（`method.md` §2）。
  3. **なぜこのメソッドか**：既存IT会社は3軸の一部しか持たない（道具特化／納品特化／上流すぎ）＝`method.md` §3。**※社名・業種を出さず役割類型で**。
  4. **具体の手触り（4つの問い）**：「AI活用したい」「担当が辞めた」「最適化したい」「自動化したい」の裏にある構造（`sales-process.md`）。**抽象論にしない核。**
  5. **協業性**：手段は専門に渡す／我々は顧客の側に立つ＝マージンを取らない（`method.md` §4）。
- **注**：4つの問いは業種非依存＝そのまま使える。専門用語は読み手に翻訳。

### ④ こんな会社へ（課題＝"状態"で括る）
- **目的**：業種を名指しせず、**読者が「うちのことだ」と自己判定**できる入口。
- **ブロック**（"状態"の例・業種非依存）：
  - 情シスがいない／一人情シスが兼任で回らない
  - 道具（システム/SaaS）を入れたが使われていない
  - 担当者が辞めて属人化が露呈した
  - 部署ごとにバラバラに導入して連携できていない
  - ベンダーと対等に話せず言いなりになっている
  - 経営は焦るが現場が動かない（言葉が噛み合わない）
- **注**：ここは ④メソッドの「4つの問い」と対。**ターゲットの絞りは出さず、状態で広く拾ってから相談で選別**する。

### ⑤ 料金の考え方
- **目的**：金額で脱落させず、かつ安売りに見せない。
- **方針（媚びない注／たたき台）**：
  - **具体額の一覧は出さない**。「**考え方＋レンジ＋無料相談で確定**」に留める（B2B伴走の通例。月額を表で晒すと相場が固定され上げられない＝HANDOFF 立ち上げ枠思想）。
  - 出すのは：**入口＝構造診断（◯万円〜・単体で価値が出る）／伴走＝月額固定／自走後＝ライト**、の3段だけ。
  - **無料相談は「診断ではない」**（在り処は見せる・解き方は診断から＝`sales-process.md`）。
- **注**：「立ち上げ枠（先着・期限つき割引・無料にはしない）」を出すかは要判断（出すと初速、出すと安く見える両面）。

### ⑥ わたしたちについて（理念・しないこと・代表）
- **目的**：一人会社の不安を、**人と哲学**で埋める。
- **ブロック**：理念（横に立って一緒に成果を出し、一緒に強くなる）／**絶対にしないこと5つ**（＝会社の輪郭）／代表の人物（手を動かす実働者であること）。
- **注**：理念文言は本人の口調へ。対句っぽさを削る。

### ⑦ 事例 / 想定モデル
- **目的**：費用対効果を具体で見せる。**初期は実績ゼロ＝"想定モデル"で代替**。
- **ブロック**：
  - **「想定モデルケース」と明記**し、固有名・業種を伏せた一般形で：**曖昧導入の出戻りコスト → 診断で回避／運用改善で年◯万の改善**（`model-case.md` の数字を一般化）。
  - 1社目ができたら **実事例に差し替え**、メソッドの"証拠"として接続。
- **注**：架空を実績に見せない。誠実さがそのまま差別化（2章②）。

### ⑧ ジャーナル（発信・信頼装置）
- **目的**：種まき・出会った後の安心。**KPIは相談につながった数のみ**（PV・フォロワーを主目的にしない＝HANDOFF 9章）。
- **ブロック**：4つの問いを読み物化／構造の見方の解説。**解き方の手の内は出しすぎない**（在り処まで）。

### ⑨ 無料相談（CTA／`lp-structure.md` §A のLP）
- **目的**：唯一のコンバージョン。**30分・相互選別**。
- **ブロック**：何を話すか（困りごと→「なぜ?」で目的に開く）／**ミニ診断にしない**／事前フォーム／申込。
- **注**：クロージングは出さない（相談の場でやる）。ここは"申し込みやすさ"だけ。

## 3. 全体トーン・運用の注意（媚びない）

- **業種・固有名を出さない分、抽象に逃げやすい**。→ ④「こんな会社へ」と③「4つの問い」の**具体の手触り**で抽象を打ち消す（ここが生命線）。
- **一人会社×事例ゼロ**：強みは「人・哲学・やり方の筋」。それをトップ→メソッド→わたしたち、で連続させる。
- **更新コスト**：ジャーナルを主軸化しない（信頼装置どまり）。サイトに時間を取られて営業の場（登壇）が止まる本末転倒を避ける。
- **メソッド名が入る箇所**：①トップ④／③メソッド／（必要なら）FV。名前確定後にまとめて差し込む。
