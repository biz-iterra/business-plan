# サービス実務の分解 — 本人がやる業務／外注する想定業務（たたき台）

> 作成: 2026-07-03 / ステータス: DRAFT v1（本人が違和感を起点に削る前提）
> 目的: ①サービスの具体（いつ・何をやるか）を実務レベルまで落とす ②各タスクを「本人コア／外注可／資産化」に分類する。これが `pricing.md`（料金の原価根拠）と、外注導入の判断材料（コンサル指針 WAIT の解除条件）になる。
> 前提: `service-definition.md`（商品の中身・契約）／`target-conditions.md`（顧客条件）／HANDOFF 13章（人員=WAIT。「本人にしかできない判断」と「外部に渡せる作業」を分解し、スポット補助から）。
> 凡例: **★＝本人にしかできない**（構造の見立て・判断・経営者との関係）／**◆＝外注可能（スポット）**／**▲＝資産化・AIで圧縮**（テンプレ・自動化＝L-3。同価格で稼働半減の源泉）

## 1. 無料相談〜契約（営業実務）

| タスク | 分類 | メモ |

| 事前フォームの確認・会社の下調べ（HP・求人・登記） | ◆素材集め → ★見立て | 下調べの「収集」は渡せる。何を意味するかの読みは本人 |
| 無料相談30分 本番 | ★ | 相互選別の関門。ここは絶対に渡さない |

| 契約書・NDA | ▲雛形 → ◆士業レビュー（スポット） | 雛形確定後は差し替えのみ |

## 3. 濃い伴走（月次ルーチン・竹の場合 月40〜50h 想定）

| タスク | 分類 | メモ |
|---|---|---|
| 週次定例（1〜2h × 月4回） | ★ | 螺旋を回す本体（学ぶ→試す→振り返る） |
| 課題ボードの運用・進捗整理 | ★判断 ＋ ▲ツール | 判断は本人。ボード自体は相手が更新できる形に（自走の練習） |
| ベンダー折衝の同席・要件の翻訳 | ★ | **価値の本体。絶対に渡さない** |
| 定量シートの月次更新 | ▲テンプレ ＋ ★解釈 | 数字の意味づけは本人 |
| 月次レポート＋経営者報告（30分） | ▲AI下書き → ★仕上げ | 経営者との接点は減らさない（財布の維持） |
| チャット相談（随時） | ★ | 応答SLAは約束しない（時給発想の侵入口） |
| 実装作業（設定・データ移行・自動化） | ◆コーダー or ▲自分でAI | 判断基準: **1回きり＝外注／繰り返す＝資産化して自分で** |
| 手順書・引き継ぎ資料づくり | ▲テンプレ ＋ ◆清書 | 「中身は手元に残す」の実体。自走の道具 |

## 4. 逓減期・ライト

- **逓減期**: 定例を隔週に半減。課題ボードは相手が回し、本人は見守り＋要所の介入のみ。→ 竹の約半分の工数（月20〜25h）
- **ライト**: 月1定例（1h）＋チャット＋四半期レビュー（半日）。→ 月5〜8h。**システム保守・実作業は含まない**（含めると代行に戻る）

## 6. 外注の設計原則（WAIT との整合・覆さない）

1. **固定費化しない**: 月額契約（伴走）1社が現金化するまで外注ゼロ。以降も件数・時間単位のスポットのみ。正社員・固定外注・営業代行はしない（コンサル指針＋HANDOFF 13章）。
2. **商品の成立に外注を組み込まない**（HANDOFF 7章）: 外注が全員消えても診断・伴走は回る。外注は「本人の時間を買い戻す」ためだけに使う。
3. **マージンを取らない**（method.md・非依存の担保）: 実装外注は実費ベースで顧客に開示。ただし**ディレクション（要件定義・検収）は自分の実工数の対価として別建てで取る**——中抜きはしないが、タダ働きもしない。
4. **渡す順番**（時間単価の低い順・失敗コストの低い順）:
   - ①文字起こし・清書・一覧化（オンラインアシスタント） — 想定 2,500〜4,000円/h・月5〜10h＝月2〜4万
   - ②図版・資料整形（クラウドソーシング／織物屋系デザイナー） — 1〜3万/件
   - ③実装（コーダー・副業の隙間稼働＝既存ネットワーク） — 想定 5,000〜8,000円/h・案件単位
   - ④請求・調整などの事務（③まで回ってから）

**媚びない注記**: §2〜3の工数を見ると、**外注で買い戻せるのは月10h前後**が当面の上限（★が大半のサービス設計だから。それ自体は非依存・属人価値の裏返しで正しい）。「外注で伴走4社目」は成立しない＝**3社上限は外注では動かない**。動かすなら「▲資産化」で自分の工数を圧縮する方が先。

## 7. ▶要記入（本人）

- 各工数目安の実感値との差（特に診断35h・伴走竹45hが体感と合うか）
- 「渡せるが渡したくない」業務（こだわり・品質の生命線）はどれか
- コーダーの実単価・受けられる作業範囲（設定系まで可か、開発のみか）
- オンラインアシスタントを使った経験の有無・抵抗感
