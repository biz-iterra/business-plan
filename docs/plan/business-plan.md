## 0. この計画書の使い方（全体像マップ）

この事業を一文で言うと——

> **「経営課題にITを活用したいのに、社内に分かる人がいない／整いきらない」中小企業に、診断→伴走→定着の長期支援で並走し、混乱した体制・仕組みを構造から立て直して、自走できる状態を一緒に作る事業。**

## 1. わたしたちが大事にすること（理念）

> ※本書に載せるのは、会社の"向き"を示す3つの柱まで。**動機の核・絶対にしないこと・判断基準は本書には載せず、MVV／コンセプト／パーパスとして別途策定する**（下の注記）。
> ※この章の言葉は最も「AIっぽさ」が出やすい。中身はそのままに、**本人の口調に必ず書き換える。**

- **何のための会社か**：クライアントの成果を一番に考え、共に伴走することで、目まぐるしい変化のある環境に適応するための強固な経営基盤を整える。
- **誰のための会社か**：経営課題にIT活用を考えている企業。そのために、実際の実務で困っている担当者、方針を決めきれない経営者、悩みを抱える組織全体に向き合う。クライアントの信頼が人を伝って作られていく関係性を構築する。
- **自社の強み**：目的に沿った課題解決のため、噛み合っていない体制や仕組みの原因を突き止め、最適な形へ導き、軌道修正とともにより良い状態にしていくこと。システム・業務・担当者を同じ「構造」としての目線で共に考えられること。

> **【本書に載せない範囲＝別途MVV等として策定】**
> エンジン（動機の核）・絶対にしないこと（5つ）・迷ったときの判断基準（P-5＋身体感覚）は、企業理念・コンセプトを形づくる"考え方"なので事業計画書には載せない。

## 2. 何を売るのか（コア事業とメソッド）

### コア事業＝支援事業

- **① コーポレートIT総合支援（主柱・現在の本体）**：IT担当者が不在／詳しくない中小企業の混乱を整理し、必要な部分だけを前に進める。経営課題をITで整える「コーポレートIT」基盤の構築支援。当面の売上も設計も、すべてここが軸。
- **② 新規事業支援（構想段階）**：新しいサービスや会社を始めようとしている企業に対し、①で貯まったノウハウ・ツール（診断手順・テンプレ・仕組み）をベースに、立ち上げ〜運用を継続的に回す「**DevOpsのような**」事業支援。①の資産の横展開カード。①が回り始めてから着手（詳細は第15章）。

### メソッドの中心

**課題を引き出し、解決に向けて改善を繰り返し、自走できる状態を作るサイクルを回して支援する。**

- 支援は**スポット対応（システムや環境整備をして終わり＝受託納品に近いもの）ではなく**、経営における2〜3年計画のように、**長期的にクライアントと共に成長する支援をコアとする。**
  - ※スポット対応は、目的が長期的な経営課題解決にならず、本質改善にならないケースが多いため。
- **提供方式**：長期の関わりの中で、螺旋を描くように**山と谷を作りながらサイクルを回す**（一定の強度で張り付き続けるのではなく、濃淡をつける）。
- **支援体制＝構造別に対応**：担当者レイヤー・組織レイヤーというように、構造の別に応じて対応する。**担当者 ＞ 経営陣 ＞ 組織全体** のように対応の目線を切り替え、**粒度にゆらぎがないようにする。**
  - ※担当者目線のまま組織全体の話をするとブレる。目線と粒度を意識的に合わせる。
  - ※これは「支援するときの目線の粒度」の話。「誰に売るか＝お金を払う人と価値を感じる人」の顧客二層は第4章で別に扱う。
- 組織全体を支援していくことで、**担当者が成長し、当社に依存しないナレッジを残すよう努め、ブラックボックス化させない。**

> ※「育成→卒業」という旧コンセプトは棄却済み（縦の関係・関係の終わりを含意するため）。本メソッドは「自走できても関係は続く（負担の軽いリテーナー契約へ）」＝**依存させないが見捨てもしない**設計。覆さない。

### 3-1. 全体ターゲット（顧客条件ベース＝主）

| # | 条件 | なぜ排除か |

### 3-2. 業界を絞る場合のターゲット（提供サービスを売るときのペルソナ例＝従）

> ここは**業種専門会社になるための絞り込みではない**。条件（M1〜M4）に合う会社が介護／小売なら普通に受ける、その際の**痛み・数字の当て方の手札**として温存する（詳細ペルソナは `archive/` に保管）。

- **介護（例）**：情シス不在のど真ん中（記録・請求・シフトが制度上必須で属人化・退職トリガー頻発）。梃子＝加算（売上）・離職コスト・監査リスク・業務時間。
- **小売・多店舗（例）**：受発注・経理・店舗間共有が紙／FAX／Excelで属人化。梃子＝在庫の機会損失・業務時間・属人化／離職・EC伸びしろ。

> ⚠️ 介護・小売の具体ペルソナ（介護の事務長兼IT担当／小売のEC・販促担当 等）と数字（特養C／HCチェーンH）は業種固有のため、条件ベースの営業トークでは「相手に合わせて出す手札」に位置づけ直す（詳細は `archive/`）。汎用モデルケースは第7章の宿題。

### 3-3. あえて狙わない会社（断る基準＝輪郭）

- 自前でできるIT企業・VC型スタートアップ（対等になりにくい）※2026-07-14 限定し直し：創業期の一人社長・個人事業主は別レーン「スタートアッププラン」（`pricing.md` §1.6・HANDOFF 18章）で受ける

## 4. お客さまは「2人」います＋勝ち筋

この事業の肝は、**お金を払う人と、価値を感じる人が違う**こと。ここを取り違えると売れない。

### 二層構造

| 層 | 誰か | ITERRAから見た位置づけ |
|---|---|---|
| **利用層（主役・最初の味方）** | 現場で実際に困っている担当者。技術的にこちらを評価する人 | 痛みを抱える当事者。**最重要** |
| **意思決定層（財布）** | 予算を握る経営者（多くは現場非出身） | 予算承認。落とせば契約 |

**すれ違いの構造**：痛みを感じる担当者には決裁権がなく、決裁する経営者は痛みを感じていない（コスト／数字でしか動かない）。だから「現場は欲しがるが経営者が動かない案件」が大量に滞留している。

### 勝ち筋（この事業の独自の武器）

> **「現場を楽にする人」ではなく、「現場の痛みを"経営が投資すべき数字"に翻訳して、経営者に決断させる人」。**

このすれ違いを埋められる人がほぼいない（ベンダー＝道具を売るだけ／コンサル＝現場を知らない／担当者＝経営の言葉を持たない）。翻訳した一枚を**Phase0診断の納品物の核**に必ず入れる（第6章「経営メリット定量シート」）。

## 5. 競合と立ち位置（4C）

### 競合（他の選択肢）と比べた位置

| お客さまが取りうる選択肢 | 足りないもの＝わたしたちの価値 |
|---|---|
| ITベンダーに頼む | 道具は入れるが業務・人の流れまで設計しない。経営数字に翻訳しない |
| 代行業者に丸投げ | 楽になるが**依存**。担当者が主役になれない |
| 経営／ITコンサル | 戦略は語るが現場を知らず絵に描いた餅 |
| 社内でIT担当を採用 | 中小は採れない・続かない・属人化（＝退職リスクの再生産） |

この4つの「欠けている部分」を一人で埋める位置に立つ。

### 4C（顧客視点）

- **顧客価値**：道具でなく「現場が自分の力で回る状態」を、経営数字に翻訳しながら依存させずに作る。
- **コスト**：月額固定で費用が読める。安さでは勝負しない。回収の見通しを先に数字で示す。
- **利便性**：基本オンライン・近郊は訪問可。最初の診断は単体で価値が完結する（買いやすい入口）。
- **コミュニケーション**：経営者には経営数字で、担当者には「あなたを主役にする」言葉で、伝え方を分ける。

## 6. 商品・サービスの設計

### 商品の「階段」

```
無料相談（売り込みでなく相互選別。30分・無料）
  ↓
構造診断（ヒアリングプラン30万／同行プラン50万〜・拠点加算／立ち上げ枠10万）
  ↓
実務伴走期（月額固定・準委任。期間は診断で見積もる）
  フル120万 ／ パートナー75万（主力） ／ アドバイザリー40万
  ↓

  ↓
リテーナー契約（チャット・ウェブ相談＋四半期レビュー・ガバナンス維持ベース。卒業させず長く続く）
```

この商品の段階を「**診断 → 伴走 → 定着**」と呼ぶ（コンサルの3段階と同型／`target-conditions.md` §4）。メソッドの型「引き出す→回す→自走」（第2章）を、この商品の段階に載せて提供する——**型は不変、段階は売り方**。定着フェーズは「運用ルール・可視化＝ITガバナンスの確立」と言語化できる。（2026-07-14 v2：対外の括り＝**ステージ1 見える化／ステージ2 実務伴走／ステージ3 保守**。プラン名・単価の正は `pricing.md`）

> **別レーン（本体の階段の外・2026-07-14 新設）**：ビジネス支援 for スタートアップ＝**スタートアッププラン 3万/月**（一人社長・個人事業主限定。チャット相談＋軽作業チケット月1件。将来は学生移管で薄利多売）。正は `pricing.md` §1.6／HANDOFF 18章。

### Phase0 構造診断（最初の有料商品・単体で価値が成立）

進め方：①契約（NDA＋業務委託）②事前ヒアリング（経営者・担当者を別々に）③**現場観察＝訪問1回以上必須**④分析⑤報告会（経営者と担当者同席）。標準2〜4週間。

**納品物4点セット**：
1. **現状構造図（As-Is）**：業務・情報・人の流れ。属人化点・二重入力点・断絶点を明示。
2. **改善マップ（To-Be＋優先順位）**：あるべき構造と効く順。道具でなく構造の直し方。
3. **伴走計画書**：期間・体制・金額・**自走の基準**（終わりはカレンダーでなく状態で定義）。
4. **経営メリット定量シート**（独自の核）：①②を経営の数字に翻訳した一枚。

> Phase0には「**お断りするための関門**」という裏の役割もある。筋が通らない／主役不在の会社には、診断を納品したうえで伴走を勧めず終える（＝「形だけ出さない」の実践）。

### 伴走の提供形態（時間で売らない）

料金は**月額固定**。「働いた時間×時給」では請求しない。理由：①時給だと「関わりが減るほど売上が減る」＝依存させない理念と衝突②顧客が時間を監視し対等な伴走者が作業者に格下げされる③速く・上手くやるほど損をして資産化の動機が消える。

→ 約束するのは稼働日数でなく **(a)接点の頻度・(b)到達すべき状態**。月の主成分は「代わりにやる」でなく「**一緒に直して現場が自分で回せる形に資産化する**」。

### 契約・権利（共通チェックリスト）

- **NDA・要配慮情報**：形だけにしない。
- **成果物の権利分離**：顧客固有（構造図・改善マップ）＝顧客／**汎用フレーム・診断手順・テンプレ＝自分に留保**（第10章「資産の再利用」の生命線）。
- 診断＝業務委託（成果物完了）／伴走＝準委任（稼働日数を約束しない）。

### 価格の運用ルール

- **立ち上げ枠：先着5社・期限つき**で診断を割引（通常30万→10万）。**無料にはしない**（1円でも払えば当事者になる）。
- **伴走の月額は割らない**（本体相場が安く記憶され後で上げられなくなる）。
- **実務伴走は同時3社まで**（品質を守る自覚的な線引き）。

## 7. 費用対効果（モデルケースの数字）

> ⚠️ **条件ベースへの転換で、業種固有のモデルケース（特養C／HCチェーンH）は"相手に合わせて出す手札"に位置づけ直した。** 業種を外した営業トークで使う**汎用モデルケース（例：多店舗／多拠点の中小企業）は未確定＝宿題**（`target-conditions.md` §6）。以下は業種例として保持。

- **出戻り＝診断の価値（考え方）**：基幹／業務システムの刷新は、要件・構造が曖昧だと出戻りで費用が倍増しうる。**診断（数十万）で数百万〜千万規模の出戻りを防ぐ**——という「将来コストの保険」で診断を売る。
- **年間損失＝伴走の価値（考え方）**：業務時間・離職／属人化・機会損失・（業種により）取りこぼし加算や在庫損失を積み上げ、改善メリットを年額で示す。
- **原則**：数字は保守的に置く（盛らない）。実案件が1つ取れたら実額へ差し替える。

## 8. どう売るか（無料相談 → 診断）

詳細は `sales-process.md`。

### 設計の軸：無料相談は「ミニ診断」にしない

> 無料相談で見せるのは課題の**「在り処」**まで。「解き方」は診断から。相手が持ち帰るのは答えでなく**「うちの問題は道具じゃなくて業務の構造だ」という気づき1つ**。

これが「無料コンサル化（消耗）」と「売り込み化（対等を失う）」の両方を防ぐ防波堤。

### 流れ

```
CTA「無料相談30分」＋事前フォーム（規模・困りごと1行）
 → 目的を握る（売り込みでなく相互に見極める場）
 → 困りごとを聞く →「なぜ？」で目的に開く
 → “構造問題”に名指し（どの入口でも「業務構造が整理されてない」に合流＝道具屋に落ちない核）
     A AI活用      → 焦り型=エスカレーション／放置型=情報共有
     B 担当退職     → 属人化を要素分解／代行は自ら断る → リテーナー契約の必然性へ
     C システム最適化 → 部署単位導入の連携欠如／ガバナンス
     D 効率化・自動化 → 無くせる作業の自動化／俯瞰役不在
 → クロージング：診断＝将来コストの保険（出戻りの数字／第7章）
     筋が通る  → まず Phase0診断一本（伴走は並列で出さない＝階段を壊さない）
     筋が通らぬ → 診断すら勧めず終える（断る関門）
```

## 10. どう儲けるか（収益構造の4層）

「法人格を持っただけのフリーランス」で終わらないため、「自分の時間」でなく「プログラム・資産・関係」を売上の主語にする。

- **第1層 プログラム型**：到達点（成果）に値段をつける。当面の現金の主柱（診断＋伴走）。
- **第2層 自走後のリテーナー契約**：負担の軽い継続課金が積み上がる。「卒業させない設計」が効く。
- **第3層 資産の再利用**：診断手順・テンプレを社内資産化し、同価格で稼働を半減（一人会社の利益エンジン。だから第6章の権利分離が生命線）。
- **第4層 1対多（1対N）**：講座・ワークショップ・診断ツール。

## 11. お金の計画（数字）

> 経営シミュレーション `deliverables/keiei_model.xlsx` と連動。詳細は `revenue-target.md`。融資用は `deliverables/jigyo-keikaku_format.xlsx`。**「要確認」は実数を後入力。**

### 会社が潰れないライン

- 主な固定費：役員報酬7万、社会保険1.1万、役員社宅（純15万）、通信5万、税理士3万(仮)ほか。
- **既存の受託収入（月40万）があるので、新事業がゼロでも会社はかろうじて黒字。**

### いちばんの制約は「会社」でなく「個人」

役員報酬を月7万に抑えるため社長個人の生活が苦しくなる。役員社宅化などで凌ぎつつ、6〜12月に残る穴は別途（海外資金等）で凌ぐ。**本人決定（2026-06-21）：役員報酬7万・貯蓄ブリッジのまま維持。Y2で報酬最適額を設計。**

### 引当（税金）

黒字でも消費税・法人税は翌年に払う。別口座へよける。役員報酬7万固定で会社に利益が溜まるため法人税が大きい → **Year2の役員報酬最適化で圧縮**（次の山）。
