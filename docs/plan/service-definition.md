## 0. このドキュメントの位置づけ

「無料相談」で在り処を見せ、「診断」で構造を診て、「伴走」で一緒に直す——この階段の**各段が単体で価値として成立し、かつ契約として破綻しない**ように中身を固める。固める核は3つ：**①商品の中身（何を渡すか）②契約条件（どう約束するか）③提供形態（どう関わるか）。**

揺らがせない設計判断（HANDOFF・persona より）：
- **伴走は「週何日」で約束しない**。時給発想に戻ると、対等な伴走者が時間労働者に格下げされ承認エンジンが逆回転する（HANDOFF 4章）。→ **接点（いつ会うか）と状態（どこまで自走したか）で定義する。**
- **診断は現場観察で最低1回の訪問を必須**にする。机上のヒアリングだけでは「在り処」を外す。
- **成果物の権利を分ける**：顧客固有のもの（その会社の構造図・改善マップ）＝顧客／**汎用フレーム・診断手順・テンプレ＝自分**。これが L-3「資産の再利用で稼働半減」の生命線（HANDOFF 9章）。
- **介護＝要配慮個人情報**を扱う前提。NDA・データの取り扱いを「形だけ」にしない（HANDOFF 2章「形だけ出さない」）。

## 1. 商品の階段（正＝HANDOFF 4章・6章の再掲）

```
無料相談（診断でなく対話・相互選別。30分無料）          → sales-process.md
  ↓
① 構造診断 Phase 0（竹30万／立ち上げ枠 先着5社 10万）   ← 本ドキュメント §2
  ↓  ※納品をもって「断る関門」。筋が通らねば伴走を勧めず終える
② 濃い伴走期（月額固定・準委任。期間は診断で確定）        ← §3
  松 110万(週3相当)＝拡張時/数量0 ／ 竹 75万(週2相当・主力) ／ 梅 40万(週1相当)
  ↓
③ 逓減期（関与が減ることを契約に明記。35万・自走移行）     ← §3.4
  ↓
④ ライト契約（月7万・相談/保守ベース。卒業させず継続）     ← §4
```
- 単価は仮置き（松110・梅40・逓減35は人件費基準が未確定）。**伴走期の月額は割らない**（相場が記憶として固定され後で上げられなくなる）。
- 品質限界＝**実運用は濃い伴走2社＋可変1枠**（3社上限・HANDOFF 4章）。

## 2. Phase 0 構造診断の実体化（本丸）

### 2.1 進め方（無料相談の後 → 納品まで）

| 段 | やること | 形式・目安 |
|---|---|---|
| 0 | 無料相談で「在り処」まで見せ、診断契約に合意 | sales-process.md §5 |
| 1 | **契約**（NDA＋診断業務委託）。事前資料の依頼 | 書面。§5チェックリスト |
| 2 | **事前ヒアリング**（経営者＋現場担当者の二層に別々で） | オンライン可・各60分 |
| 3 | **現場観察（訪問・必須1回以上）** | 半日〜1日。§2.3 |
| 4 | 構造の分析・4点セット作成 | 持ち帰り |
| 5 | **報告会**（経営者と担当者同席。経営メリット定量シートで意思決定に橋渡し） | 60〜90分 |
| 標準期間 | 合意〜報告会まで **2〜4週間**（会社規模で前後） | |

> 設計の肝：**②で痛みを言語化 → ③で現実を裏取り → ⑤で経営者を数字で動かす。** ヒアリングだけ・訪問なしは「在り処を外した整いすぎた診断」になり信頼を失う。

### 2.2 診断で「聞く」項目（▶を型で埋めたたたき台）

二層で別々に聞く。**担当者には"業務の実態"、経営者には"数字と判断"**を聞き分ける。

**A. 現場担当者に聞く（業務の実態・属人化の在り処）**
- いま「自分しかできない」業務はどれか。引き継ぎ書はあるか／更新されているか。
- 1日・1週・1ヶ月で**必ず発生する締め切り業務**は何か（国保連請求＝月末／LIFE提出／処遇改善加算の記録／シフト作成）。
- 同じ情報を**2回以上入力している**箇所はどこか（二重入力＝統合の在り処）。
- ベンダー／ツールは何を使い、**どこで話が噛み合わず止まるか**。
- 「これさえ無くなれば」と思う作業トップ3。
- 直近で**ヒヤリ（請求漏れ・提出遅れ・データ消失）**があったのはどこか。

**B. 経営者に聞く（数字・判断・恐れ）**
- 直近の**離職**：誰が抜けて何が止まったか。採用・教育にいくらかかったか。
- 取りこぼしている／取りこぼしそうな**加算（＝売上）**はあるか。
- **監査・実地指導**でいちばん不安な書類・記録は何か。
- ITに**年いくら**使っているか。何に対して払っている認識か。
- 「いま投資すべきか」を**何を見て**判断したいか（社長が腹落ちする一枚の条件）。

> ▶要記入：上記のうち「介護で実際にいちばん深い痛み」（請求／LIFE加算／シフト／二重入力のどれか。本人の母ルート裏取り宿題と連動）が分かったら、その問いを最前列に並べ替える。

### 2.3 診断で「見る」項目（現場観察・訪問必須）

聞き取りの自己申告と現実のズレを、目で取る。
- **実際の入力画面・台帳・Excel**を横で見る（自己申告の手順と実作業の差）。
- 紙とデジタルの**境目**（どこで紙に落ち、どこで再入力されるか）。
- 「できる人」の机まわり（付箋・自作マニュアル＝属人化の物証）。
- 情報の**置き場所と権限**（共有フォルダの構造・誰が何を見られるか＝AIが効かない根因）。
- フロアの動線・誰が誰に聞きに行くか（エスカレーションの実態）。

### 2.4 納品物4点フォーマット（HANDOFF 4章「3点セット＋定量1枚」）

| # | 納品物 | 中身 | 渡す相手 |
|---|---|---|---|
| ① | **現状構造図（As-Is）** | 業務・情報・人の流れを1枚に。属人化点・二重入力点・断絶点を明示 | 顧客固有→顧客 |
| ② | **改善マップ（To-Be＋優先順位）** | あるべき構造と、効く順（インパクト×実現性）。**道具でなく構造の直し方** | 顧客固有→顧客 |
| ③ | **伴走計画書** | 期間・体制・金額・**自走の基準**（カレンダーでなく状態で終わりを定義） | 顧客固有→顧客 |
| ④ | **経営メリット定量シート（核）** | ①②を経営の数字に翻訳（§2.5）。報告会で経営者を動かす一枚 | 経営者 |

> ②③で使う**分析フレーム・図のテンプレ・診断チェック表は自分の資産**（顧客には記入済みの成果物を渡すが、空フォーマットは渡さない）。L-3資産化の線引き。

### 2.5 経営メリット定量シートの計算ロジック（★積み残しを埋める核）

勝ち筋＝**「現場の痛み」を「経営が投資すべき数字」に翻訳して経営者に意思決定させる**（persona §3.5）。梃子は4本。**補助金は梃子に使わない**（採択型はトラブル源・2026-06-23方針）。助成金は要件を満たせば受給できる範囲で「自己負担を下げる補足」に留める。

各行は「現状の損失（年額）」→「改善後」→「差額＝メリット」→「投資（診断＋伴走）」→「回収月数」で組む。

| 梃子 | 現状損失の出し方（年額） | 必要な入力値 |
|---|---|---|
| **① 業務時間（人件費）** | 対象業務の月時間 × 時給 × 12（＋残業割増） | 月時間／時給 |
| **② 離職コスト** | 年間離職人数 × 1人あたり採用・教育・引継ぎ費 × 属人業務停止リスク係数 | 離職人数／1人コスト |
| **③ 取りこぼし加算（売上）** | 取得漏れ・返戻の加算額 × 12（請求精度↑で回収） | 月あたり漏れ額 |
| **④ 監査・返戻リスク** | （想定指摘額・返戻額）× 発生確率 | 想定額／確率 |

**計算式（テンプレ）**
```
年間損失合計  L = ①人件費損 + ②離職損 + ③加算取りこぼし + ④リスク期待値
年間メリット  B = L × 改善率r（保守的に r=0.3〜0.5 で置く。盛らない）
投資総額      I = 診断費(10〜30万) + 伴走費(月額×伴走月数)
回収月数      T = I ÷ (B ÷ 12)
（助成金がある場合のみ）実質投資 I' = I − 助成金額 → T' で併記
```
> 設計の媚びないルール：**改善率rは楽観に振らない**（HANDOFF「楽観値へ戻さない」）。盛ると報告会で崩れる。むしろ「保守的でもこれだけ回収できる」を見せる方が経営者は動く。数字の単位・桁は§7のダミー見本と LP⑦・メソッドの費用対効果と**共通の架空モデルケース**を使う（次タスク「架空モデルケースの数字」と連動）。

### 2.6 Phase0 の価格・契約条件

- **竹 30万／立ち上げ枠：先着5社 10万**（**無料にしない**＝1円でも払うと当事者になる）。松50万〜は現ターゲット規模では除外。
- 立ち上げ枠の対価3つ：**事例公開許可／率直なFB／良ければ紹介1社**（即決は迫らない・期限つき枠なら期限は伝える）。
- 契約形態＝**業務委託（成果物＝4点セット納品で完了）**。準委任でなく**請負寄り**（成果物が定義できるため）。
- 診断納品は**「断る関門」**：筋が通らない／螺旋を回す主役がいない会社には、納品をもって伴走を勧めず終える。

## 3. 濃い伴走期の提供形態（「週何日」で約束しない）

### 3.1 定義の軸＝接点 × 状態（時間で売らない）

「週2日」と言った瞬間、顧客は時間を監視し始め、伴走者が時間労働者に落ちる（HANDOFF 4章）。だから**約束するのは稼働日数でなく、(a)接点の頻度・(b)到達すべき状態**。

- 契約に書くのは「**月◯回の定例＋随時の相談対応＋このフェーズで到達する状態**」。
- 「週2相当（竹75万）」は**社内の工数見積り・値付け根拠**としては持つが、**顧客への約束文言にはしない**。

### 3.2 伴走で「月に何をするか」（★積み残し・接点設計のたたき台）

竹（主力・月75万）の標準リズム：

| 接点 | 頻度 | 内容 |
|---|---|---|
| **定例ミーティング** | 月2〜4回（隔週〜毎週） | 改善マップの実行レビュー・次アクション・詰まりの解消 |
| **現場同席／観察** | 月1回以上（要訪問の局面） | 標準化・権限整理が現場で回るかを目で確認 |
| **担当者の壁打ち（随時）** | 非同期・チャット | ベンダー対応の翻訳・判断の後ろ盾（"丸腰で矢面"を解く） |
| **資産化作業** | 毎月 | その月の改善を**手順書・テンプレに落とす**（属人化を解く＝商品の本体） |
| **経営者への月次レポート** | 月1回 | 進捗を**経営数字で**返す（定量シートの実績更新） |

> 月の主成分は「代わりにやる」ではなく「**一緒に直して、現場が自分で回せる形に資産化する**」。代行に寄ると依存を作り、HANDOFF 2章「依存させない」に反する。
> 松（週3相当）＝改革テーマが複数並走する規模で接点を増やす（現状は本人1名のため数量0）。梅（週1相当）＝テーマが1本に絞れた会社の定例中心。

### 3.3 ポテンシャル診断の組み込み（信頼ができてから）

- Phase0には入れない（初対面で人の話に踏み込むと「占いで人事を語る怪しい人」HANDOFF 4章）。
- **濃い伴走期に信頼ができてから**、「対話の共通言語・配置の設計図」として持ち込む。配置の最終判断は現実の行動・成果データで行い、診断は前菜（形だけにしない）。

### 3.4 逓減期 → 自走 → ライト

- **逓減期（35万）**：関与が減ることを**契約に明記**。接点を月1へ落とし、判断を顧客側に移す。終わりは**自走の基準**（例：定例で出る課題を自社で解けている／手順書が更新され続けている）を満たしたとき。
- 自走基準を満たしたら **④ライト契約**へ（卒業させない設計＝低稼働×継続課金が積み上がる L-2）。

## 4. ライト契約（月7万・相談/保守ベース）

- 内容：月1回の相談枠＋随時の軽い質問＋年数回の構造点検。
- 位置づけ：**卒業させない継続関係**（HANDOFF「育成→卒業」コンセプトは棄却済み。自走後も繋がり続ける）。
- 解約は妨げない（対等）。ただし「困ったら戻れる相談先」を残すのが介護の人手不足には効く（sales-process パターンB＝ライト契約の必然性に直結）。

## 5. 共通契約チェックリスト（全プラン横断）

着手前に必ず確認する。**形だけにしない**項目に ★。

- [ ] ★ **NDA**：要配慮個人情報（利用者の介護・健康情報）に触れる前提を明記。範囲・保管・返却/破棄・再委託禁止。
- [ ] ★ **個人情報の取り扱い**：診断中に利用者の実データを「見るが持ち出さない」原則。観察はマスキング/匿名化、画面は預からない。
- [ ] ★ **成果物の権利分離**：顧客固有の成果物（構造図・改善マップ）＝顧客に帰属／**汎用フレーム・診断手順・テンプレ・スキル＝自分に留保**（L-3資産化の生命線）。契約書に一文で明記。
- [ ] **契約形態**：診断＝業務委託（成果物完了）／伴走＝準委任（月額固定・稼働日数を約束しない文言）。
- [ ] **再委託**：実装外注が発生する場合の扱い（販管費「外注費」。顧客の事前同意）。
- [ ] **賠償・免責**：助言提供であり最終意思決定は顧客。成果保証はしない（がFBは率直に）。
- [ ] **期間と終了**：伴走はカレンダーでなく**自走の基準**で終了判断する旨。逓減期の関与減を明記。
- [ ] **対価3つ**（立ち上げ枠のみ）：事例公開許可・率直なFB・良ければ紹介1社。

## 6. Phase0 納品物4点のダミー見本（★積み残し・架空モデルケース「特養C」）

> 数字は仮（次タスク「架空モデルケースの数字」で LP⑦・メソッドと共通化して確定する）。ここでは**フォーマットの見本**として置く。

**架空ケース：特別養護老人ホームC（従業員60名・情シス不在・事務の佐藤さんに業務集中）**

### ①現状構造図（As-Is）見本
```
[利用者情報] ─手入力→ [介護ソフト] ─手集計→ [請求(国保連)]   ← 月末・佐藤さん属人
     │                     │
     └─紙台帳─→ [Excel(シフト)]   [LIFE提出] ←勘で運用・要件未翻訳
                     ↑
           情報共有フォルダ＝権限バラバラ → AIも検索も効かない
断絶点：①請求が佐藤さん1人 ②同じ利用者情報を3箇所に二重入力 ③LIFE要件が誰も分からない
```

### ②改善マップ（To-Be＋優先順位）見本
| 優先 | 改善（構造） | 効くもの | 道具より先に直すこと |
|---|---|---|---|
| 1 | 利用者情報の**入力一元化**（二重入力を断つ） | 時間・ミス | 入力責任と項目定義を決める＝構造 |
| 2 | 請求業務の**手順書化＋複数人化** | 属人化・離職リスク | 「人を増やす」でなく標準化で分散 |
| 3 | 情報フォルダの**権限・構造の再設計** | AI/検索が効く地面 | 道具の前に置き場と権限 |

### ③伴走計画書 見本
- 体制：本人1名／竹（月75万）／**到達状態**：請求が2人で回り手順書が更新される・二重入力ゼロ。
- 期間目安：濃い伴走 約6ヶ月 → 逓減3ヶ月 → ライト（自走基準＝定例課題を自社で解けている）。

### ④経営メリット定量シート 見本（§2.5の式で。数字の正＝`archive/model-case.md`）
| 梃子 | 現状損失/年（保守） | 改善率r | メリット/年 |
|---|---|---|---|
