## 0. このプロジェクトの性質（最初に読む）

- ゼロベースで「事業の理念・コンセプト・商品・収益構造」を設計している。**他案件の文脈は持ち込まず、まっさらで考えている。**
- 進め方は「たたき台を出す → 本人が違和感を起点に削る/直す」の反復。**本人は、整いすぎた言葉・借り物の概念・忖度を嫌う。** 抽象論より、根本原因まで掘った具体を好む。
- Claudeへの期待は「媚びない・合理と感情の両面で率直に伝える壁打ち相手」。同意できない点は同意しないこと。

## 1. 確定した理念（5本柱）

### P-1 何のための会社か
**横に立って伴走し、一緒に成果を出し、一緒に強くなる。**
代わりに全部やる代行屋でも、裏方でもなく、**対等な並走**。相手が自分でできるようになるまで伴走する。

### P-2 誰のための会社か
顔の見える相手。現場で困っている担当者と、その人を抱える組織。匿名の大量ではなく、**信頼が人から人へ連鎖する**関係。

### P-3 自分の強み
噛み合っていない構造を根っこまで掘り、筋の通った形に作り直せること。学んだことを知識で終わらせず、現実で動かせること。**システム・業務・人の配置を、同じ目で構造として見る。**

### P-4 エンジン（動機の核）
自分の力が現実で機能し、担当者が主役として輝き、**その担当者から評価される**——という二段構えの「強い承認」。
→ だから「相手を主役にする」のは道徳ではなく、**本人の欲望と一致した戦略**。

### P-5 判断基準（迷ったときに自分に聞くこと）
- 全体として筋が通っているか？（単一の側面＝金だけ・情だけで決めていないか）
- 現実で本当に機能するか？ 根っこに効いているか？
- 相手が主役か？ 対等に向き合えているか？
- これをやって、自分も伸びるか？

## 2. 絶対にしないこと（＝会社の輪郭）

1. 相手を「自分なしじゃ回らない」状態にしない（依存させない）
2. 見た目だけ整えて、実際は動かないものを納品しない
3. 相手の手柄を横取りしない（主役は相手）
4. 単一の側面だけで仕事を決めない（金だけ・情だけ。ただし総合的な合理性の中で金も情も軸に入れてよい）
5. スキルの高さを「偉さ」や序列にしない（社内外問わず・対等。媚びず・見下さず）

**※ なぜ重要か:** 本人は外から押し付けられたルールに従えない人。だが「自分の価値観から自分で導いた絶対にしないこと」だけは守れる。だからこれは本人にとって唯一機能する自前のルール。新セッションは、この5つを上書き・緩和する提案をしないこと。

## 3. コンセプトの核（重要な転換あり）

### サービスOS：引き出す → 回す → 自走
当初の「育成 → 卒業」を**棄却**した。理由（覆さないこと）:
- 「育成」は縦の関係（先生と生徒）。本人の「対等」原則と矛盾する。
- 「卒業」は関係の終わり。本人の「信頼が連鎖し続ける」エンジンと逆向き。

代わりに、相手の中にあるポテンシャルを**引き出し**、成長サイクルを一緒に回す。

### 成長サイクル（螺旋）
**学ぶ → 試す・実践する → 振り返る・可視化する →（一段上がって）また学ぶ……**
- 円ではなく**螺旋**。一周ごとに少し高い位置に戻る。その積み上がりが「成長」。
- 「成長」はサイクルの1ステップではなく、サイクルを回した結果＝外側の目的に置く。
- 螺旋が相手の力で回り出した状態＝「自走」。卒業させなくても自走と継続的関係が両立する。

### 二層構造
- **個人の層（エンジン）**：学ぶ→試す→振り返るの螺旋を一緒に回す。これが価値の本体。
- **組織の層（土台）**：ポテンシャル診断による共通言語とチーム配置。個人のループが回る地面を整える。
- **売り方の結論**：toB か toC かは偽の二択。「**組織に売って、個人に効かせる**」。金は組織から、意味は個人へ。心の置き場所（個人）と財布の向き（組織）は分けてよい。

### ポテンシャル診断の扱い（注意）
- 生年月日ベースの独自診断。科学的実証性では戦えない前提で扱う。
- **Phase 0（構造診断）には組み込まない。** 初対面で信頼が乏しい段階で人の話に踏み込むと「占いで人事を語る怪しい人」になるリスク。
- 濃い伴走期に入り信頼ができてから、「対話の共通言語・配置の設計図」として持ち込む。
- 「形だけのものを出さない」原則を診断自身にも適用。配置の最終判断は現実の行動・成果データで行い、診断は前菜であって主菜にしない。

## 4. 商品設計

### 領域（同じサービスOS上のアプリ）
- **A：情シス伴走**（本丸・継続収益）— 情シス不在/一人情シスの中小企業の担当者と並走
- **B：開発プロジェクト伴走**（入口・スポット高単価）— ベンダーと対等に話せない会社のPM代行的伴走
- **C：バックオフィス×IT**（筋が通る場合のみ）— 代行に寄りやすいので選別

### 商品の階段
```
無料相談（診断でなく対話・相互選別）
  → 構造診断 Phase 0（竹30万 / 松50万〜）
    → 濃い伴走期（月額固定・準委任。期間は診断で確定）
      → 逓減期（関与が減ることを契約に明記）
        → ライト契約（月5〜10万・相談/保守ベース）
```

### Phase 0 構造診断の中身（単体で価値が成立すること）
- ヒアリング（経営者＋担当者）＋現場観察＋IT棚卸し＋人の構造把握
- 納品3点セット：① 現状構造図(As-Is) ② 改善マップ(To-Be＋優先順位) ③ 伴走計画書（期間・体制・金額・**自走の基準**）
- 裏の機能：本人が「断る」ための関門。筋が通らない/螺旋を回す相手がいない会社は、診断納品をもって伴走を勧めずに終われる。

### 期間について
**6ヶ月/12ヶ月は仮置き。** 固定するのは期間ではなく**構造**（濃い伴走→逓減→ライト、終わりはカレンダーでなく「自走の基準」で判断）。各フェーズ長は会社ごとに Phase 0 で見積もる。

### 課金形態の決定：月額固定（準委任）一択
**「工数×時給」は採用しない。** 理由（覆さないこと）:
- 時給は「関与が減るほど売上が減る」＝依存させない原則と直接矛盾
- 顧客が時間を監視し始め、対等な伴走者が時間労働者に格下げされる
- 速く・うまくやるほど損をする＝資産化のインセンティブが消える
→ 売るのは時間でなく「状態の変化」。工数は内部の原価管理に格下げ。

### 同時クライアント上限：3社（品質保証の方針）
**なぜ:** 3社を超えると、各案件の文脈を思い出すだけで工数がかかり、伴走の質が落ちる（本人の実測体感）。これは弱点でなく品質限界の自己認識。実運用は「濃い伴走2社＋可変1枠」。可変枠は常に埋めない（資産化・営業・休息の緩衝材）。

### 立ち上げ枠（戦略的例外）
- **先着5社・期限つき：構造診断を割引（例 通常30万→10万）。無料にはしない**（1円でも払うと当事者になる）。
- 値引きの対価を必ず取る：事例公開の許可／率直なFB／良ければ紹介1社。
- **伴走期の月額は割らない**（本体の相場が記憶として固定され、後で上げられなくなるため）。
- これは売上でなく「実績と信頼連鎖」への投資。

## 5. 収益構造（時間売りからの離脱）

「法人格を持ったフリーランス」で終わらないための4層。売上の主語を「自分の時間」から「プログラム・資産・関係」へ移す。
- **L-1 プログラム型**：到達点に値段をつける（現金の主柱）
- **L-2 自走後のライト契約**：低稼働×継続課金が積み上がる（卒業させない設計が効く）
- **L-3 資産の再利用**：診断手順・テンプレ・スキルを社内資産化しながら納品。同価格で稼働半減（一人会社の利益エンジン）
- **L-4 1対N**：講座・ワークショップ・診断ツール。映像/イベント人材のネットワークが活きるのはここ

## 6. 経営・財務（会計レベルまで落とし込み済み）

### 確定した数字の前提
- 損益分岐（当初の理解）：個人の生活費 月50万
- 既存受託（当面の土台）：月40万
- 滑走路：海外資金の引き出しで **約4ヶ月分（200万）**
- 価格：診断 立ち上げ10万/通常30万・松50万〜、濃い伴走 月70〜80万/社、ライト 月5〜10万/社

### 会計レベルで見えた重要事実
1. **会社の真の損益分岐は月約68万**（役員報酬50万＋会社負担社保7.5万＋販管費10万）。既存受託40万で埋める差は月10万ではなく**約28万**。ただし伴走1社（月75万）で軽く超える。
2. **税金の罠**：初年度は黒字でも、消費税（簡易課税・サービス業みなし50%）＋法人税等で**約256万**が翌期に出ていく「自分の金ではない現金」。これを別口座に避けないと、黒字なのに納付月に詰まる（一人法人の典型事故）。**インボイス登録済み＝初年度から課税事業者。**
3. （慎重シナリオでも）負の連鎖は止まる。税引当後の「本当に使える現金」の底でもプラスを維持できる試算。

### 役員報酬の方針（本人決定）
- **立ち上げ年は役員報酬を月7万に下げる。** 生活は貯蓄で橋渡し。
- 効果：会社の損益分岐が**月約18万に激減**→既存受託40万だけで会社は即黒字。**会社は初日から潰れない。**
- **本当の制約が「会社」から「個人」に移る**：生活費50万−報酬7万＝月43万の不足を、貯蓄200万で**約4.6ヶ月**もたせる勝負。会社と個人は別の財布。
- 注意：定期同額給与のため期中変更は不可。立ち上げ年は7万で固定し、新規売上が立って利益が貯まったら **Y2で増額**。
- 注意：報酬を極端に下げると会社利益が増え法人税も増える。立ち上げ期の現金温存としては正しいが、軌道に乗ったら「法人税＋所得税＋社保」の合計が最小になる報酬額に最適化する。

### 営業ファネル（受注に必要な活動量）
- 変換率の仮置き：アプローチ→無料相談 15%、無料相談→診断 35%、診断→伴走 50%
- 結果：診断1件を取るのに **月約19件のアプローチ＝営業時間 月約9.5h**。週1日の営業で到達可能。
- **この変換率の実数化が、モデル精度の次の鍵**（紹介中心かコールドかで大きく変わる）。

## 8. リスク（数字で扱えないもの）

- **代替不能性**：本人が倒れると伴走が止まる。対策＝現金の厚み＋協力者との関係維持。
- **承認エンジンの逆回転**：認められたい欲が強いため、評価してくれそうな相手の「筋の通らない案件」を受けてしまう瞬間が来る。→ 判断基準（P-5）に戻る。
- **風呂敷の拡大**：人材送り込み→転職斡旋→非IT展開→診断×配置…と発想が広がりやすい。**今 金を払う相手に実証済みなのはITの伴走だけ。** 新概念は「構想フォルダ」に置き、まずITで1事例を作る。
