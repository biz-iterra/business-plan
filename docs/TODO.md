## 0. なぜこの順番か（媚びない前提）

ご依頼は大きく2系統：**①数字（PL/BS・月次必要売上）** と **②事業の中身（ターゲット/ペルソナ/痛み/メリデメ/ベネフィット/4C）**。

直感では「数字が先」に見えるが、根っこを掘ると逆。

- **「月いくら売上が要るか」の骨は、既存モデルがもう答えている。**
  - 役員報酬7万固定 → 会社の損益分岐 **月約18万**（既存受託40万だけで黒字）
  - 役員報酬50万（軌道後） → 損益分岐 **月約68万**
  - 締め切り＝個人金庫200万 ÷ 月不足43万 ＝ **約4.6ヶ月**
- 未確定なのは単価×成約率で、それは **「誰に・どの痛みを売るか」が決まらないと仮の数字にすらならない。**
- だから**ターゲット/ペルソナ/痛みを先に尖らせ → その実態で前提セルを上書き → PL/BS/Y2Y3を仮で完成**、が筋。

→ **推奨順序：①→②→③（下記フェーズ）。** ただし「必要売上ラインの明文化(T8)」と「BSの箱作り(T7前半)」は既存モデルから先行着手可能。

## 各タスクの掘り方（中身メモ）

### T1 ターゲット
- HANDOFFのざっくり「情シス不在/一人情シスの中小」を、**業種 × 規模(売上/従業員/IT予算) × トリガー状況(いつ最も困るか)** で1セグメントに絞る。
- 絞る基準＝P-5（筋が通る／根っこに効く／相手が主役になれる／自分も伸びる）＋3社上限の品質方針。
- 「断れる関門」を効かせるため、**あえて狙わない層**も明記する。

### T2/T3 ペルソナと痛み
- 二層：①現場の担当者（主役・エンジンが効く相手）②財布を握る経営者。痛みは別物として分離。
- 痛みは表面（例：ベンダーと話が噛み合わない）→ 根っこ（例：要件を翻訳できる人が社内に居ない構造）まで。借り物の痛みを書かない＝本人の実体験/受託の現場から引く。

### T4 メリデメ・ベネフィット
- メリットだけでなく**デメリットも正直に**（依存させない＝相手も汗をかく必要がある、即効性は低い等）。
- ベネフィット＝痛みが消えた後の状態。機能(コスト削減)でなく**担当者が主役になる/経営者が夜眠れる**まで。

### T5 4C分析 ※定義の確認待ち
- 解釈A（マーケ4C）：Customer Value／Cost／Convenience／Communication（4Pの顧客視点裏返し）
- 解釈B（環境分析4C）：Customer／Competitor／Company／Co-operator（3C＋補完者）
- いずれでも**競合（代行屋・ベンダー丸投げ・社内採用・他コンサル）との差別化**は必ず含める。

### T6 前提の見直し
- 対象セル：単価 `前提!C11-C14`、ファネル率 `前提!C29-C33`。
- T1/T3で見えた実態に寄せる。**楽観値へ戻さない**。動かしたら HANDOFF 6章へ「いつ・なぜ・覆さない条件」を追記。

### T7 PL/BS
- **BSは現状モデルに無い → 新規シート。** 箱：資産(現金/売掛/未収) 負債(未払消費税/未払法人税) 純資産(資本金/利益剰余金)。
- PLをY2/Y3まで会計レベルに拡張（消費税・法人税・均等割を年次で）。
- 鉄則：**前提の黄色セル連動・数式を壊さない。openpyxlで書く→LibreOfficeで再計算→値を検算。**

### T8 必要売上ライン
- 既存モデルから逆算：各月の最低必要売上を、損益分岐(18万/68万)・砂時計(4.6ヶ月)・税引当と重ねて「いつまでに伴走何社」を月次で明文化。
