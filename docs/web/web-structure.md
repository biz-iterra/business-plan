## 0. 会社の事業構造（2026-07-12 本人決定・HANDOFF 16章）

3事業は横並びでなく**①から②③が生まれる川の流れ**。売上は約8割が①想定。②③は「①の副産物」＝非依存の哲学と整合（客が成果物・ノウハウを持って自走できる）。

```
① ビジネス支援事業（ITERRA hands-on-support）  ← 主軸・厚く作る
   ├─ コーポレートIT総合支援（伴走〜自走〜ライト）
   └─ アセスメント（構造診断）＝単体で完結する入口
        └→ 現場で生まれた ノウハウ→② ラーニング（Academy） ／ 成果物→③ コンテンツ販売
```

- **重心は①。②③は「ある」レベル（薄く）。** ②③は今後やめる可能性あり＝作り込まない（覆さない条件・HANDOFF 16章）。
- **メソッド（ITERRAメソッド）は①専用。** ②③をメソッドで正当化しない。
- 診断を独立させても**診断→伴走の階段リンクは消さない**。

## 2. トンマナ（実装から確定・全ページ共通）

### デザイン（実装済み＝これがひな型）
- **色**：基調 **soleil**（太陽）。`/journal` 配下のみ sage。terra=会社・sage=クライアントの三色系（corporate-iterra CLAUDE.md）。
- **型**：`PageHeader`（日本語タイトル＋EN小ラベル＋パンくず）→ `Section`（EN小ラベル併記）を積む。部品は DS（`@iterra-inc/ui`）：GuideCard／FeatureList／StepsProcess／ComparisonTable／PricingCard／Accordion／Table。**新ページもこの部品で組む（新規部品を作らない）。**
- **書体**：Noto Sans JP＋EN見出しアクセント。余白広め・1行の長さは `.mw-40em` 系で制御。

### 文体（実装のLPトーンを正とする）
- **です・ます。断定は静かに**（煽らない・盛らない）。リードは1〜2文。
- **“ ”で概念を引用強調**（例：“なんとなく困っているIT”）。数字は正直、架空モデルは「※架空のモデルケースです」を必ず併記。
- 一人称＝**わたしたち**（ひらがな・LP実装準拠）／読み手＝**あなた**。※`/company/` の「私たち」と不統一 → 統一は本人判断。
- **語彙のキーワード5**：コーポレートIT／構造／翻訳／伴走／自走。
- **禁止**：業種名（介護・小売 等）・ペルソナ名／①の文脈での「代行します」（哲学と自己矛盾）／空語（DX推進・最先端・革新）／依存を誘う語（まるごとお任せ）／煽り（今すぐ・限定※立ち上げ枠非掲載の決定で不要に）。
- FVの切り口＝**実装済みイントロ「“なんとなく困っているIT”を、経営の数字で動かす伴走者。」を採用ベース**とする（旧FV4案の A構造×B翻訳ミックスが実装で選ばれた形。最終語調は本人）。
